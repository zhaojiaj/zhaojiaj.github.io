欢迎使用

欢迎来到Charles。这个文档是个一个正在进行中的工作。如果你在这没有找到你的答案，请联系我们。

Charles目标是成为一个潜入软件。开启这个软件，，它会尝试自动配置你的浏览器，所以你能马上准备好并且开始运行。开始浏览web并且查看再Charles中返回的结果；点击他们并且观察被记录了什么。

如果要继续阅读，请看右边的目录列表部分。

安装

安装Charles

首先下载Charles，下载地址->charles.如果你使用Firework，请同时下载Firework插件。

Windows

运行安装程序，把Charles安装在你的程序菜单中。

Mac OS X

双击下载文件，将其解压缩，然后复制Charles应用程序复制到到你的应用程序文件夹中。

Linux

Charles具有APT和YUM的存储库，如果您具有基于Debian或基于Red Hat的Linux发行版，则这是安装Charles的首选方式。

否则，将tar.gz归档文件解压到一个合适的位置。如果你之前已经安装过Charles并且正在执行更新操作；首先确保Charles不在运行状态，然后在以前安装的头文件进行安装或者复制。通过运行bin/Charles脚本开启Charles。

开始使用

请查看开始使用指导（下一部分）来开始使用Charles！

APT存储库

Charles有一个用于基于Debian的Linux发行版的ATP存储库。

注：为了利用更大的密钥和更强的汇编，，回购协议的密钥已于2016.7.26发生了改变。ATP存储库中已经出现的用户需要导入新的公共密钥。新的公共密钥和旧的公共密钥在同一个URL地址，因此重复下面的apt-key添加步骤来添加它。最后运行apt-get update来运行新的密钥。

首先为存储库安装GPG公共密钥，因此你可以确认这个软件包是否签名正确。
现在的公钥id是AD28806并且它的指纹是4BA7 DB85 7B57 0089 7420 96E1 5F16 B97C 1AD2 8806:

wget -q -O - https://www.charlesproxy.com/packages/apt/PublicKey | sudo apt-key add -

或者可替换为：

sudo apt-key adv --keyserver pgp.mit.edu --recv-keys 1AD28806

然后把存储库安装到你的资源中：

sudo sh -c 'echo deb https://www.charlesproxy.com/packages/apt/ charles-proxy main > /etc/apt/sources.list.d/charles.list'

然后更新你的源代码并且安装Charles：

sudo apt-get update
sudo apt-get install charles-proxy

这个包在/user/bin中创建了一个“charles”命令，并且把Charles添加到 窗口管理器的应用程序菜单中。

你也可能安装Charles的一个被称为charles代理测试版的测试轨道。

APT存储库

使用charles-proxy3的安装包来安装Charles 3。你可以同时安装Charles 4和Charles 3。

这个包在/user/bin中创建了一个“charles3”命令，并且把Charles添加到窗口管理器的应用程序菜单中。

YUM存储库

Charles有一个用于基于Red Hat的Linux发行版的YUM仓库。

cat <<EOF > /etc/yum.repos.d/Charles.repo
[charlesproxy]name=Charles Proxy Repository
baseurl=https://www.charlesproxy.com/packages/yum
gpgkey=https://www.charlesproxy.com/packages/yum/PublicKey
EOF

然后安装Charles：

sudo yum install charles-proxy

这个包在/user/bin中创建了一个“charles”命令，并且把Charles添加到窗口管理器的应用程序菜单中。

你也可能安装Charles的一个被称为charles代理测试版的测试轨道。

（注：charles-proxy-beta、charles-proxy-beta、Red Hat、Red Hat需了解）

准备开始

学习怎样使用Charles和它可以怎样帮助你最好的办法是安装它并且观察他是怎样工作的。如果你还没有准备好请看上方安装的部分。

开始使用Charles

通过你的“开始”菜单、“应用程序”文件夹或者任何你安装Charles的地方运行该应用程序来启动Charles。

如果你已经安装了Firefox，你将会被询问一个关于Firefox附加组件的问题。这个让你告诉Charles你是否已经安装了Firefox附加组件或者没有安装。

在Mac OS X上你将被会问到一个关于你的Mac OS X代理设置的问题。为了能够在Mac OS X自动配置你的代理设置你需要通过输入密码来授予Charles权限。

你可以对这两个问题回答“还没有（Not Yet)”，稍后完成后你将获得开始使用Charles的机会。

第一印象

你最先看到的东西是一个未命题的会话。如果你的代理设置已经被自动配置，你现在应该可以使用你的web浏览器并且观察已经被记录在Charles中的事件。

你会看到你所访问的每一个网站/主机的节点都显示在左边的树中。探索这棵树。点击某一个已经被记录的事件，你会在正文（body）中看到详细信息（可能仅仅在响应的选项卡上，除非你正在执行POST），并且你会看到正在传输的数据。

这是你可以用Charles做的最简单、最普通的事。它具有强大的功能，因为你可以看到正在传输和接收的每一个内容——它使你可以深入了解实际发生的事情。

Hexo & Github博客搭建教程

关键词

  • Hexo
  • 静态博客
  • Github pages
  • hexo theme（主题）

步骤

1.安装Node.js

下载地址->node.js

安装完成后，按win+R打开命令符，输入node -v和npm -v，如果能出现版本号，就说明你安装成功啦。

2.Git与GitHub

Git->下载地址

安装完成后，按win+R打开命令符，输入git –version，如果能出现版本号，就说明你安装成功啦。

下载完Git，你当然还需要一个GitHub账号（记得去注册哟），用来存放网站.

···Github新建博客仓库···
--------------------------------------------------

注意：仓库名必须是GitHub的用户名+github.io！！！

···Github Pages···
--------------------------------------------------

项目建成后，点击Settings，向下拉到最后有个GitHub Pages，

注意：是新建项目的settings哦!

点击Choose a theme选择一个你喜欢的主题，再回到GitHub Pages。点击链接就可以看到你刚才选择的那个主题啦。

3.安装Hexo

在电脑合适的位置新建一个文件夹存放网站，本文中将其取名为blog
在文件夹根目录下右键，找到git bash here,依次输入如下指令

npm i hexo-cli -g #-g表示全局安装

hexo -v #验证是否安装成功

hexo -init #初始化

hexo install #安装依赖包

npm install #安装所需组件

hexo generate #生成静态网页

hexo server #在本地服务器运行

你也可以通过命令hexo s -p xxxx #把默认端口号修改

执行完最后一步，会返回一个网址（默认端口号是4000），用浏览器打开，如果出现下图的页面，就说明你成功了一半，如果你不喜欢这个主题，后面part将会讲到如何更换主题↓↓↓

（如果执行完上面的命令，没有返回网址，出现下面的情况，你可以回到你的blog文件下，查看有没有下面这些文件，如果没有说明你前面执行命令的过程中出现了差错，重新初始化将上面那些命令重新执行一遍即可。）


详情可查看next文档！！!

本地搭建好后，下面我们需要

···Github与本地连接···
--------------------------------------------------

右键打开git bash，然后输入下面命令

git config --global user.name "zhaojiaj" “ “内的账号为刚才GitHub注册的帐号名字

git config --global user.email "1165465022@qq.com" “ “内的邮箱为刚才GitHub注册的帐号邮箱

···生成密钥SSH key设置到Github···
--------------------------------------------------

同样，还是右键git bash here,输入下列命令：

cd ~/.ssh

连续三个回车，你会得到以下三个文件

找到文件夹C:\Users\lenvo.ssh(地址可能有所不同)生成的文件，用记事本打开***.pub的文件，复制内容到GitHub->settings->ssh and gpg key->new ssh sky中（记得要新建一个ssh key哦，名字可以随便起）

如下就是成功了！

···本地配置···
--------------------------------------------------

找到blog文件夹下_config.yml，用sublime打开(记事本也可)

按照如下修改，repository修改为你自己的github项目地址

记住repository冒号后面要有个空格，不然会报错！！！

···用Git上传···
--------------------------------------------------

根目录下输入右击git bash here

hexo g生成静态网页

hexo s预览本地效果

hexo d上传到github上

或者直接输入hexo g -d作用就是部署、上传,一键操作。

注：这几个命令是常用命令，以后写完博客或者更新配置后都会用到

配好后，如果出现让你登陆的页面，就说明配置成功啦！

这时打开你的 ****.github.io主页就能看到前面提到的hexo默认的主题啦

···上传文章···
--------------------------------------------------

在根目录blog文件夹下右击git bash here

hexo new "文件名"

在_posts中会有新建的文章，用markdownpad2打开编辑即可。(一般默认自动新建文章名字为hello-world.md）

左边编辑文章内容，右边可以同步展示效果。

如果第一次下载使用markdownpad2打开md文件时可能报错，那是HTML渲染错误。

因为缺少了组件，安装Awesomium 1.6.6 SDK即可

http://markdownpad.com/download/awesomium_v1.6.6_sdk_win.exe

文章编辑好后，再次回到根目录下git bash here

hexo g -d #部署上传 再次打开主页就可以看到你写的文章啦

4.Hexo theme

官网有许多主题的源码你可以下载到自己的电脑上，下载地址->hexo主题

下面就介绍一下切换主题的方法：

首先从上面的链接进入hexo theme主页

然后选择一套你喜欢的主题进入作者主题的GitHub仓库，下载主题的压缩包

把下载的到本地的压缩包解压，可以把解压文件夹的名字改成简短的，复制粘贴到blog/themes下

接下来一步修改配置，_config.yml文件下拉到最底下，把theme后主题的名字改成你刚才clone到blog/themes中主题的名字

最后，hexo g -d #部署上传

主题就修改成功啦！

但是，如果你有强大的前端基础或者对前端感兴趣，也可以尝试自己写一个主题，参考->hexo模板
